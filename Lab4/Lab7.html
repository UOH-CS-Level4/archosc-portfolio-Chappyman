This lab focused on CPU scheduling and how operating systems manage processes. C# was used to implement scheduling algorithms such as First-Come-First-Serve (FCFS) and Round Robin, while calculating finish time, turnaround time, and normalised turnaround time for different processes. Gantt charts were also created to visualise how processes were scheduled, and different time slices were tested to analyse their impact on system performance and context switching.

1. Priority Scheduling

Priority scheduling is a CPU scheduling algorithm where each process is assigned a priority value. The CPU is given to the process with the highest priority first. Processes with the same priority are usually handled using First Come First Served (FCFS). Priority scheduling can be preemptive, where a higher priority process interrupts the current process, or non-preemptive, where the running process finishes before another begins. One disadvantage is starvation, where low priority processes may never get CPU time if higher priority tasks keep arriving.

2. Shortest Job Next (SJN)

Shortest Job Next (SJN), also called Shortest Job First (SJF), is a scheduling algorithm that selects the process with the smallest execution time first. It is mainly used in batch systems and helps reduce average waiting time. SJN can be efficient when process burst times are known in advance. However, long processes may suffer from starvation if shorter jobs continue to arrive.
