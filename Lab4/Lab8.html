Lab 8 focused on memory management in operating systems, including virtual memory, paging, page tables, and translation lookaside buffers (TLBs). Different tasks involved calculating virtual and physical addresses, identifying page faults, and understanding how the MMU translates memory addresses. C# was also used to simulate TLB hits and misses, while later tasks explored memory interfacing and how processors communicate with memory using decoders and control bits.

Task 1

(i) Determine the number of virtual pages and bytes per page?

With a 4-bit VPN: number of virtual pages = 2⁴ = 16 pages
With an 8-bit offset: bytes per page = 2⁸ = 256 bytes

(ii) A load instruction generates virtual address 0x2C8. What is the VPN, offset and corresponding PPN?

0x2C8 in binary = 0010 1100 1000
Split into [4-bit VPN | 8-bit offset]:
VPN = 0010 = 2
Offset = 11001000 = 0xC8
Looking at VPN 2 in the page map: R=1, D=0, PPN=4
Physical address = (4 × 256) + 200 = 0x4C8

Task 4.1

(i) How many pages reside in physical memory at one time?

2²⁴ ÷ 2¹⁰ = 16,384 pages

(ii) How many entries are in the page table?

2³² ÷ 2¹⁰ = 4,194,304 entries

(iii) How many bits per entry in the page table?

14 (PPN) + 1 (resident) + 1 (dirty) = 16 bits

(iv) How many pages does the page table occupy?

4,194,304 × 2 bytes = 2²³ bytes → 2²³ ÷ 2¹⁰ = 8,192 pages

Task 4.2

What is the physical address for virtual address 0x1804? Explain the MMU process.

0x1804 in binary = 0001 1000 0000 0100
Offset (lower 10 bits) = 0x004
VPN = 6
VPN 6 is found in the TLB (R=1, D=1, PPN=2) → TLB hit, no page table lookup needed.
Physical address = (2 × 1024) + 4 = 0x804
The MMU checks the TLB first. Since VPN 6 is present and resident, it reads PPN=2 directly and appends the offset, returning 0x804 without touching the page table.

What happens when virtual address 0x0FC is generated?

0x0FC → VPN = 0, Offset = 0xFC
VPN 0 exists in the TLB but R=0, meaning the page is not in physical memory. This triggers a page fault — the OS loads the page from disk, updates the page table and TLB, then retries the instruction.
